# Algorithm: What is an Algorithm and Pseudocode

Before writing any code, you need to understand the blueprint. An algorithm is that blueprint — a step-by-step plan for solving a problem. Think of it as the recipe your program follows.

## What is an Algorithm?

An algorithm is a finite set of instructions that, when followed, will solve a specific problem. It must be clear, unambiguous, and eventually terminate. You already use algorithms every day without realizing it.

## Real-World Examples

Making a cup of tea is an algorithm:

1. Boil water
2. Put a tea bag in the cup
3. Pour boiling water into the cup
4. Wait 3 minutes
5. Remove the tea bag
6. Add milk or sugar if desired

Getting directions to a friend's house is another one. Each step leads to the next until you arrive.

## What is Pseudocode?

Pseudocode is a way to write algorithms in plain English that looks like code but does not follow any specific language's rules. It lets you focus on logic rather than syntax.

Pseudocode uses keywords like `IF`, `ELSE`, `FOR`, `WHILE`, and `PRINT` to describe what happens, but the rest stays in natural language.

## Writing Pseudocode

Here is a simple algorithm to check if a number is even or odd:

```
INPUT number
IF number MOD 2 equals 0
  PRINT "Even"
ELSE
  PRINT "Odd"
END IF
```

Notice how it reads almost like English. That is the whole point — clarity first.

## Why Pseudocode Matters

Pseudocode forces you to think through the problem before touching a keyboard. It catches logic errors early and serves as documentation that anyone can read, regardless of which programming language they know.

> 💡 Tip: Always write your algorithm in pseudocode first. It is faster to fix mistakes on paper than in code, and it helps you see the big picture.
